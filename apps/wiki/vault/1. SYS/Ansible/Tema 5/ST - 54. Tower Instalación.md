# Requisitos:
- Servidor específico para <span style="color:rgb(167, 110, 212)">Tower</span> (No compartir con otras aplicaciones) {serv.web y serv.bbdd}
- Distribuciones soportadas a partir de:
	- Red Hat Enterprise Linux 7
	- Centos 7
	- Ubuntu 14.04 / 16.04
- Navegadores como Chrome, Brave, Firefox, etc.
- Al menos <span style="color:rgb(167, 110, 212)">2 GB</span> de <span style="color:rgb(167, 110, 212)">RAM</span> (Recomendado <span style="color:rgb(167, 110, 212)">4 GB</span>)
- Al menos <span style="color:rgb(167, 110, 212)">20 GB</span> de <span style="color:rgb(167, 110, 212)">disco</span>, de los cuales <span style="color:rgb(167, 110, 212)">10 GB</span> deben ser dedicados a <span style="color:rgb(255, 0, 0)">/var/</span>.
- En el caso de <span style="color:rgb(167, 110, 212)">RHEL</span>/<span style="color:rgb(167, 110, 212)">CentOS</span> es necesario habilitar repositorio <span style="color:rgb(167, 110, 212)">EPEL</span>.
# Escenarios:
1. <span style="color:rgb(167, 110, 212)">APP</span> y <span style="color:rgb(167, 110, 212)">BBDD</span> juntos en el mismo server.
2. Un <span style="color:rgb(167, 110, 212)">server</span> para la <span style="color:rgb(167, 110, 212)">APP</span> y otro <span style="color:rgb(167, 110, 212)">server</span> para la <span style="color:rgb(167, 110, 212)">BBDD</span> (Postgres 9.4.x).
