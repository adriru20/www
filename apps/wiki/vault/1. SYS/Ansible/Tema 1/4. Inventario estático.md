# Formato
## Grupos:
``` yaml
[Grupo]
Server1
Server2
```
## Subgrupos:
``` yaml
[Subgrupo:children]
Grupo1
Grupo2
```
## Variables:
- **Jerarquía**: Host-> Grupo-> GrupoPadre
``` yaml
[Grupo:vars]
ansible_become=True
```

Para separar las variables por grupos y hosts, se establecen las siguientes rutas:

| Grupos                            | Servidores                          |
| --------------------------------- | ----------------------------------- |
| /etc/ansible/group_vars/**grupo** | /etc/ansible/host_vars/**servidor** |
# Sintaxis
Podemos poner ***[1:5]*** o ***[a:b]*** al poner los servidores para establecer rangos, es decir:
- Un rango de IP: **192.168.1.*[2:4]* ansible_user=ansible**
De la misma manera podemos establecer distintos ficheros de configuración copiando el fichero hosts para separar en entornos de DEV, QA y PROD (Se suelen almacenar en el directorio /etc/ansible/inventarios/*fichero*). Usaremos `ansible -i fichero all` para poder ejecutarlo.
# Variables
``` Bash
ansible_connection {
	local
	ssh {
		ansible_host
		ansible_port
		ansible_user
		ansible_ssh_private_key_file
	}
}

ansible_become {
	False
	True {
		ansible_become_method {su || sudo}
		ansible_become_user #Por defecto a root, pero podemos poner otro usuario si queremos.
	}
}
```
