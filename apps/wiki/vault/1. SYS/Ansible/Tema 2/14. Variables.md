# Posibles usos:
- Definir el puerto de una aplicación.
- Indicar un usuario.
- Establecer una ruta de fichero.
- Pueden ser:
	- Playbook (Definirlas dentro)
	- Facts (Obtenidas del servidor)
	- Línea de comandos (`-e`)
- Se pueden usar en
	- Tareas
	- Plantillas (Hosts)
	- Otros (condiciones, bucles, etc)
# Ejemplo:
Creamos un template llamado hosts.j2 con el siguiente contenido:
```yaml
{{ miip }} {{ ansible_hostname }} {{ ansible_fqdn }}
```
Y un playbook llamado variables.yml con el siguiente contenido:
``` yaml
- name: Crear fichero usando variables
  hosts: localhost
  connection: local
  vars:
    - miip: '1.2.3.4'
  tasks:
    - name: Crear fichero hosts
      template: src=hosts.j2 dest=/temp/hosts
```
Al ejecutar `ansible-playbook variables.yml` y hacer un `cat /tmp/hosts`, nos mostrara el siguiente contenido: `1.2.3.4 centos9 centos9.adriru.es.`
Con el siguiente comando podemos ver las claves de las variables que vayamos a usar:
``` Bash
ansible localhost -m setup | grep -e ansible_hostname -e ansible_fqdn 
```
