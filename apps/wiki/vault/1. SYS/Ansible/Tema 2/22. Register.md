Esto nos puede servir para almacenar la salida de los comandos lanzados y luego mostrarla con un debug.
Ej. práctico
``` yaml
--- #salida.yml
- name: Mostrar salida de un comando
  hosts: python
  become: False
  tasks:
    - name: Ejecutar uptime
      command: uptime
      register: salida_uptime
    - name: Salida del uptime del host python
      debug: var=salida_uptime.stdout
```
Los valores que devuelve son:
``` yaml
ok: [python] => {
    "salida_uptime": {
        "changed": true,
        "cmd": [
            "uptime"
        ],
        "delta": "0:00:00.004105",
        "end": "2025-08-16 15:22:28.709166",
        "failed": false,
        "msg": "",
        "rc": 0, #Si pone 0 es porque el comando se ha ejecutado correctamente.
        "start": "2025-08-16 15:22:28.705061",
        "stderr": "",
        "stderr_lines": [],
        "stdout": " 15:22:28 up  1:01,  1 user,  load average: 0,00, 0,00, 0,00",
        "stdout_lines": [
            " 15:22:28 up  1:01,  1 user,  load average: 0,00, 0,00, 0,00"
        ]
    }
}
```
Los valores que nos pueden devolver son:
- **Changed**: Cambio de permisos a administrador.
- **Failed**: Fallo al ejecutar la tarea.
- **Skipped**: Salta la tarea, podemos forzarlo añadiendo al final de la tarea un `when: 1==0`.
- **rc**: Que nos indica si el comando ejecutado se ha ejecutado correctamente o no.
- **stdout** / **stdout_lines**: Salida normal.
- **stderr** / **stderr_lines**: Salida de errores.

Solo mostrar cuando el fichero se ha modificado:
``` yaml
--- #salida.yml
- name: Mostrar salida de un comando
  hosts: python
  become: False
  tasks:
    - name: Ejecutar uptime
      command: cat /etc/hosts
      register: salida_uptime
      when: copiado|changed      #Solo se mostrará cuando se realice un "changed" y guardaremos la salida en la variable copiado
    - debug: var=copiado.stdout  #Esto nos mostrará (en el caso de que se realice un "changed", por el when: de abajo) el resultado por consola.
      when: copiado|changed
```
También podemos realizar lo siguiente para ver el contenido del fichero modificado:
``` yaml
--- #salida.yml
- name: Mostrar salida de un comando
  hosts: python
  become: False
  tasks:
    - name: Ejecutar uptime
      command: cat /etc/hosts
      register: salida_uptime
      when: copiado|changed      #Solo se mostrará cuando se realice un "changed" y guardaremos la salida en la variable copiado
    - debug: var=salida_uptime.stdout_lines  #Esto nos mostrará el contenido del fichero por consola.
```
